# Effective / Expert Java

In this workshop, attendees learn how to write Java that is correct, readable, and maintainable — moving past "code that compiles" to code that holds up under change, scale, and a team. The focus is on the judgment that separates senior engineers: when to reach for a feature, when not to, and why.

The workshop is modular and tailored to meet the specific needs of your team. We begin with a complimentary assessment to understand your goals and expectations, ensuring the content is aligned with your needs.

Key learning points:
- designing APIs that are hard to misuse: minimal surface, clear contracts, fail-fast
- immutability by default, and the real cost of mutable shared state
- `equals`/`hashCode`/`compareTo` done right (and why getting them wrong is so expensive)
- generics, bounded wildcards, and type-safe heterogeneous designs
- exceptions: checked vs unchecked, designing for recoverability, not swallowing failures
- `Optional`: where it belongs and where it doesn't
- effective use of the Stream API — and when a plain loop is the better call
- value semantics: records, identity vs equality, and value-based classes
- encapsulation, package boundaries, and minimizing accessibility
- allocation, boxing, and hot-path costs — without premature optimization
- defensive copying, builders, and safe construction
- common foot-guns: nulls, mutable statics, leaky abstractions, over-inheritance
- composition over inheritance in practice
- testing for correctness: contracts, edge cases, and meaningful coverage
- code review as a design tool: what to look for, what to let go
